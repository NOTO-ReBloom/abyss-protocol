# ABYSS PROTOCOL REV.28 QA REPORT

## 対象

- `index.html`
- `game.html`
- `analyst.html`
- `ABYSS_PROTOCOL_MANUAL.pdf`
- 章解放・物語進行・任務前後のストーリー表示

## 実施した検査

### 1. JavaScript構文

`game.html` のメインスクリプトを抽出し、Node.jsの構文検査を実施。構文エラーはありませんでした。

### 2. 章解放

- 初回起動時：第1章のみ選択可能
- 第2章から第5章：未解放・選択不可
- 第1章クリア後：第2章を解放
- 解放状態：`localStorage` の `abyssStoryUnlocked` と任務記録から復元
- 途中章を飛ばして解放しないことを確認
- 進行リセット操作で、第1章のみの状態へ戻ることを確認

### 3. 物語表示

任務前に以下の4項目が表示されます。

1. 前章までの物語
2. 現在の状況
3. 二人の間にあるもの
4. 今回の任務

任務中は装置解除ごとに、章固有の記録断片・会話・物語段階が更新されます。

クリア後は以下を表示します。

- この章で判明したこと
- 湊と澪の関係の変化
- 次に残った疑問
- 新たに解放された章
- 回収した記録断片

失敗時も物語の進行は失われず、再挑戦時の現在地を表示します。

### 4. 設定整合性

旧版に存在した以下の矛盾表現を除去しました。

- 湊が三年間施設内に取り残されていたという表現
- 湊が死亡扱い・死亡者の声として扱われる表現
- 遙が現在も生存して通信しているように読める表現

REV.28では、湊は2093年の事故から生還し、2096年に自らA-13へ再進入します。遙の音声は生存者ではなく、事故前に残された案内モデルです。

### 5. 画面収まり

ブラウザ自動操作による確認結果：

- 1600×900：任務選択画面、ゲーム画面、結果画面でページスクロールなし
- 1366×768：任務選択画面、ゲーム画面、結果画面でページスクロールなし
- 結果カード内の縦横オーバーフローなし
- 実行時JavaScriptエラーなし

詳細数値は `browser_qa_rev28.json` に記録しています。
